# resume-context：回專案的第一動作

**原則：恢復脈絡階段全程唯讀，禁止修改任何檔案、禁止跑會改變狀態的指令。**

## 步驟（依序執行；該工具/檔案尚不存在時跳過並註明）

1. 讀 `PROGRESS.md` 的 🧭 快速回憶區（五欄 + 上次收工日期）。
2. `git log --oneline -10` 與 `git status --short`（尚未 git init 則跳過）。
3. `git tag --list "phase-*"` 確認最後收掉的 Phase。
4. `uv run pytest -q` 確認現狀可信（尚未建 uv 環境則跳過）。
5. `nvidia-smi` 確認 GPU 可用（Phase 1+ 需要抽關鍵點時才必要）。
6. 讀 `docs/PLAN.md` 第 6 章「當前 Phase」的 DoD 清單，對照哪些已勾。
7. 若快速回憶區「未決問題」有條目，讀 PLAN.md 第 2 章 Decision Log 確認是否已有後續決策。

## 回報格式（三段，繁體中文）

- **現況**：最後收工日期、目前 Phase、pytest 結果、最後一個 tag。
- **下一步**：快速回憶區「下一步」那條指令/動作（原文照引），加上你對照 DoD 後的補充。
- **阻塞**：「待使用者人工處理」欄的未完成項、以及任何測試紅燈/環境異常。

回報完等使用者指示才動工。若快速回憶區與 git log/測試結果矛盾（例如寫著全綠但 pytest 紅），如實指出矛盾，以實測為準。
